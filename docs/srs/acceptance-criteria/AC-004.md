# AC-004

## Criterion

A student can join a live quiz using a valid 6-digit code and reach the appropriate state (waiting / active / finished).

## Linked Requirements

- FR-QUIZ-001 (code entry)
- FR-QUIZ-008 (name input constraints)
- FR-QUIZ-009 (topic preview with +N)
- FR-QUIZ-010 (waiting room)

## Verification Steps

1. Admin creates a session with status `waiting`.
2. Open `/`. Tap `Gabung dengan kode`.
3. Enter the 6-digit code. Field auto-strips non-digits.
4. Topic preview card renders MAPEL / BAB / SUB rows with `+N` chips.
5. Type a name (≤ 24 chars, raw casing kept).
6. Tap `Gabung`. `join_player_to_quiz` RPC fires.
7. Routed to `/quiz/<code>` waiting room with bouncing dots and a countdown card if `expires_at` is set.
8. Mount picker appears. Pick a preset or build a custom skin (mount + jersey/pants/saddle hex).
9. Admin starts the quiz. Realtime channel flips status to `active`. Runtime opens.

## Expected Result

The system validates the code, registers the player, persists `quiz_player_<code>` in encrypted localStorage, and routes to the waiting room or runtime based on `session.status`. Joining mid-game lands on the current global index and skips passed questions.
