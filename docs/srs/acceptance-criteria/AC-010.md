# AC-010

## Criterion

An admin can add, edit, delete, hide, and batch-hide questions.

## Linked Requirements

- FR-QM-005 (add question)
- FR-QM-006 (edit question, permission-aware)
- FR-QM-007 (delete question, permission-aware)
- FR-QM-008 (toggle visibility)
- FR-QM-009 (batch visibility)
- FR-QM-010 (rich text editing)
- FR-QM-015 (sanitization)

## Verification Steps

1. Sign in as admin.
2. Open Questions tab. Tap `Tambah soal`.
3. Fill in: question text (rich text with bold + KaTeX inline math), MAPEL/BAB/Sub-bab (multi-select), type = `multiple_choice`, options A–E, pick correct answer.
4. Save. The new row appears in the table.
5. Tap edit on the new row. Change the question text. Save. Confirm the change persists after a tab refresh.
6. Tap the visibility toggle on a row. Confirm `is_hidden` updates and the row tone changes.
7. Select multiple rows via checkboxes. Tap `Sembunyikan` (or `Tampilkan`). Confirm `BatchVisibilityConfirmModal` opens.
8. Confirm the modal. All selected rows update.
9. Tap delete on a row. Confirm the destructive modal opens.
10. Confirm delete. Row disappears.

## Expected Result

All CRUD actions persist via RPC. DOMPurify sanitization is applied on save and render. KaTeX, tables, code blocks, and `<img>` tags survive the sanitizer. Permission-gated actions (`question:update:any` vs `:own`) behave correctly when tested with a non-owner admin.
