# AC-003

## Criterion

A student can start survival mode and see lives, score, and per-answer feedback.

## Linked Requirements

- FR-EXAM-001 (mode picker, hides count)
- FR-SURV-005 (per-answer feedback popup)
- FR-SURV-006 (surrender flow)
- FR-SURV-007 (end conditions)
- FR-SURV-008 (score page layout)

## Verification Steps

1. Open `/`. Pick `Survival` in the mode segmented control.
2. Confirm the question count picker is hidden.
3. Pick categories + time limit. Tap `Lanjutkan`.
4. Step 2 shows `Survival` mode chip in red and a lives indicator.
5. Tap `Start exam`. Runtime opens with header strip showing `Skor` and `Nyawa`.
6. Pick a wrong answer.
7. The feedback popup appears with red halo + ping ring + `−1 nyawa` chip. Lives count drops.
8. Pick a correct answer. The popup is green with `+1 skor` chip. Score increments.
9. Repeat until lives reach zero (or until time runs out).
10. The session ends and the score view shows correct count, lives left (`0`), time used, with tone-aware tint.

## Expected Result

Survival forces strict navigation, displays lives in the runtime, gives instant per-answer feedback, and ends the session cleanly when lives hit zero or the timer expires.
