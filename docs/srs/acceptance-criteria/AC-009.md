# AC-009

## Criterion

A non-admin authenticated user cannot execute admin server actions or RPCs.

## Linked Requirements

- NFR-SEC-002 (admin authorization)
- FR-ADMIN-008 (permission matrix)

## Verification Steps

1. Sign in as a non-admin Supabase user (e.g., an anonymous student session).
2. Capture the access token.
3. Call an admin server action (e.g., `saveQuestionAction`) with the token.
4. Call an admin RPC (e.g., `delete_topic`) with the token.
5. Inspect the response.

## Expected Result

The server checks `admin_profile` for the authenticated user id, finds no row, and rejects the call with `Forbidden`. No data is mutated. The error message does not leak admin schema details.
