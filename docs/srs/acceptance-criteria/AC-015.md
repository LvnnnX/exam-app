# AC-015

## Criterion

The admin Results tab supports filters, pagination, detail view, and live tracking with realtime updates.

## Linked Requirements

- FR-RES-001 (view history)
- FR-RES-002 (paginate)
- FR-RES-003 (filter)
- FR-RES-004 (stats)
- FR-RES-005 (detail modal)
- FR-RES-006 (live sessions)
- FR-RES-007 (refresh)
- FR-RES-008 (live tracking)

## Verification Steps

1. Sign in as admin. Open Results tab. Confirm History view loads.
2. Confirm the four stats cards (Attempts, Avg score, Pass rate, Avg duration) render with tinted accents.
3. Apply MAPEL filter. Table updates.
4. Apply mode filter (Survival). Table updates.
5. Change rows-per-page selector. Pagination strip updates.
6. Tap a row. ResultDetailsModal opens with candidate name, topic, time range, score, and per-question breakdown.
7. Switch to Live mode toggle.
8. Confirm in-progress sessions list with answered count, lives, progress bar, start time.
9. Tap a live row. TrackingModal opens.
10. Confirm the modal subscribes to a unique Supabase Realtime channel and shows current question + history.
11. Submit an answer as that player in another browser. Confirm the modal updates without manual refresh.
12. Tap the Refresh pill in the header. Both History and Live refetch.

## Expected Result

History pagination, filtering, stats, and detail modal work as specified. Live mode subscribes to realtime updates with a polling fallback at low frequency. The Tracking modal uses a unique channel name (`player_answers_<id>_<ts>_<random>`) so multiple modals do not collide.
