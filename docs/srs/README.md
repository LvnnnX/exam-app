## Purpose

This SRS defines the current requirements for the Smandapura Exam App. Use it to guide development, testing, maintenance, and future feature work.

## How to Read

- Start with [02 Overall Description](02-overall-description.md) for context.
- Skim [03 Architecture](03-architecture.md) for the moving parts.
- Read the [functional requirements](functional-requirements/README.md) for what each feature must do.
- Cross-check [07 Non-Functional Requirements](07-non-functional-requirements.md) and [08 Business Rules](08-business-rules.md) when making design decisions.
- Use the [16 Test Matrix](16-test-matrix.md) before each release.

## Maintenance

The code is authoritative. When the SRS and the code disagree, fix the SRS. Keep this folder in sync with shipped behavior — out-of-date SRS is worse than no SRS.
